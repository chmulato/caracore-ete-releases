ASSETS SOB docs/assets/

Finalidade: folhas de estilo, scripts e media referenciados pelas páginas HTML em docs/.


SUBPASTAS USUAIS

css/
  laboratorio.css — tema escuro partilhado por várias páginas do portal.
  portal-responsive.css — camadas responsivas carregadas depois de laboratorio.css onde aplicável.

js/
  main.js — quando uma página referencia scripts locais.

img/
  Imagens usadas directamente pelos HTML em docs/ sem duplicar toda a árvore global do repo (exemplo: capturas do protótipo de degustação).

pdf/
  Manual de iniciação e outros PDF versionados junto do portal.

README.md nesta pasta resume convenções para contribuidores web.


CAMINHOS NAS PÁGINAS

Um HTML em docs/ usa normalmente prefixos do tipo assets/css/..., assets/img/..., assets/pdf/...

Algumas páginas podem apontar para outras árvores conforme histórico do documento; conferir sempre o HTML antes de mover ficheiros.


DEPLOY

O workflow deploy_site_ete_releases pode copiar assets/images da raiz do caracore-ete para docs/assets/images no repositório ETE-releases quando essa pasta existir no checkout. Manter caminhos nos HTML coerentes com o que foi efectivamente publicado.
