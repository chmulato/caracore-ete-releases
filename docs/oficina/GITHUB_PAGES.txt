PUBLICAÇÃO DO SITE HTML

URL de referência do produto: https://ete.caracore.com.br/

Este repositório (caracore-ete) mantém o conteúdo fonte sob docs/. O GitHub Pages que serve o domínio público deve estar configurado no repositório de destino para onde o workflow copia os ficheiros, não necessariamente aqui.


FLUXO AUTOMÁTICO

Ficheiro: .github/workflows/deploy_site_ete_releases.yml

Dispara em push às branches main ou master quando mudam ficheiros sob docs/ ou index.html na raiz do repositório caso exista. Também aceita workflow_dispatch manual.

Passos relevantes: clone do repositório chmulato/ETE-releases com um token, cópia do conteúdo de docs/ para docs/ nesse repositório, cópia opcional de assets/images da raiz deste repo para docs/assets/images no destino quando a pasta existir, commit e push no destino.

Secret obrigatório no caracore-ete: TOKEN_DELIVERY_CHMULATO_ETE_TO_REALEASES com permissão de escrita no repositório ETE-releases.


CONFIGURAÇÃO NO REPOSITÓRIO DE DESTINO

No GitHub do repositório chmulato/ETE-releases: Settings, Pages, Source Deploy from a branch, pasta docs na branch publicada (main ou master conforme política desse repo).
